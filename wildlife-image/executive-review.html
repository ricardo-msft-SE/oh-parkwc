Initiative Snapshot

What each use case solves and where value is expected.

UC1: Contract Processing Automation

PDF -> Excel -> Memo
  • Current pain: Manual extraction and copy/paste steps across PDF, Excel, and memo drafting.
  • AI pattern: Document Intelligence extraction, GPT-4o memo drafting, Power Automate routing.
  • Expected outcome: Significant time reduction and improved consistency for contract documentation.
  • Primary KPI targets: >= 70% time reduction, >= 90% extraction accuracy, memo quality >= 4/5.

UC2: Wildlife Image and Video Recognition

Triage + Species Tagging + Routing
  • Current pain: Very high volume visual review with many non-wildlife false triggers.
  • AI pattern: Custom Vision filtering, AI Vision tagging, Video Indexer event extraction.
  • Expected outcome: Faster wildlife event detection and major reduction in manual triage.
  • Primary KPI targets: false-trigger F1 >= 0.88, >= 60% review-time reduction, precision >= 85%.

Delivery Roadmap

Four-phase execution model with UC1 and UC2 progressing in parallel.

Weeks 1-4

Phase 1 - Scope and Baselines

Confirm pilot boundaries, baseline metrics, data inventory, and provision core Azure resources.

Weeks 4-9

Phase 2 - Data and Model Preparation

Label contract/image datasets, tune extraction/classification quality, and finalize pipeline designs.

Weeks 8-16

Phase 3 - Build and Integrate

Deploy production pipelines, automation flows, dashboards, and Copilot-assisted review workflows.

Weeks 14-20

Phase 4 - Validate and Scale

Conduct UAT, measure KPI achievement, refine prompts/models, and prepare expansion plans.

Shared Platform and Governance

Common architecture foundation to reduce duplication and improve controls.

Shared Technical Stack

  • Azure AI Foundry hub with separate UC1 and UC2 projects.
  • ADLS Gen2 as centralized archive for contracts, images, and videos.
  • Azure Data Factory and Power Automate for ingestion and orchestration.
  • Power BI, Azure Monitor, and App Insights for visibility and performance management.

Security and Compliance Controls

  • Microsoft Entra ID RBAC and segmented access for research vs. law-enforcement data.
  • Encryption at rest and in transit with secure secrets in Azure Key Vault.
  • Purview-supported lineage, retention, and governance controls.
  • Audit logging and human-review checkpoints for sensitive classifications.
